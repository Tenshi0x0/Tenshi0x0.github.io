Brief Explanation of Binomial Inversion

Intro

Consider nnn labeled elements.

Let fnf_nfn​ denote the number of scenarios in which exactly nnn elements satisfy the condition (where condition depends on the specific problem), and gng_ngn​ denote the number of scenarios in which the specified nnn elements satisfy the condition. Then it is clear that there are

Here gng_ngn​ denotes the total number of programs that specify iii elements from nnn elements to satisfy the condition.

The first one is more commonly used in a combinatorial sense.

Application

Generally the question asks for something related to fff (e.g., asking **just **how many elements satisfy the condition, at least how many elements satisfy the condition, etc.).

Often fff is not easy to find directly, so we can first find ggg and then invert the binomial to get fff.

Example

Description

In combinatorial mathematics, a derangement is a permutation of the elements of a set in which no element appears in its original position. In other words, a derangement is a permutation that has no fixed points. (From https://en.wikipedia.org/wiki/Derangement). You are then required to find how many of all permutations of nnn elements are derangement.

Solution

Here we can specify the condition as element on its original position and find $g_i = C_n^i \times (n-i)! $ (because after specifying iii elements in their own positions the other elements can make arbitrary decision positions). Finally using binomial inversion to find f0f_0f0​ is the answer.
